It is important that you do the exercises in order, even the reference exercises.  The later exercises use things you learn in the earlier ones.

If you are a beginner, be sure to do the exercises numbered 000-499 in each section.  The exercises will make more sense if you do them in order:  later exercises build on concepts and topics covered in earlier ones.  Exercises numbered 500 and above can sometimes include topics and language elements that have not yet been covered and the number 500+ exercises can safely be postponed until later if they are confusing.  If you do the 000-499 exercises in each section, you should be able to safely move on to the next section and make good progress.


C++ comes from the language C, or to put it more formally, C++ is a superset of C.  You need to know the basics of C in order to move on to the newer ideas that C++ provides about how a programming language should work.


It turns out that the way you design a program in C is pretty different from the way you do it in C++.  Many people who want to learn C++ wonder if they should "learn C first."  The answer is "yes, and no."  Yes, you need to learn the basic commands, variable types, syntax, and so on that were originally designed in C because C++ uses all of that.  But if you want to learn C++, you do not first need to program strictly in C for a long time and then "step up" to C++ when you feel you have mastered C.  You can do that if you want, but it is not necessary to master C first in order to learn C++.


Many people like the C programming language, and still use it today.  The Linux kernel is written in C.  There is nothing wrong with learning C and being a C programmer, if that is what you want to do.  But FeetWetCoding is intended to help you learn C++.


This learning framework is to be used in conjunction with a C++ textbook or other educational coursework.  Someone who was very determined might be able to figure out how C++ works just from these exercises, but trying to do it that way would be tremendously more difficult than if you have a good C++ book to refer to.  There have already been many great C++ books written, and it is not the goal of the FeetWetCoding project to reproduce everything contained in those books.


What this means is that FeetWetCoding does not explain everything about C++.  The exercises assume you have read, or have access to, some kind of C++ reference material.  Here in this help documentation, we provide links to online C++ (and other) references such as:

  • http://www.cplusplus.com/reference/


The first thing you must do if you have not already is to purchase the following book.  It is indispensible.  If you do not have K&R on the shelf, you are not a Real Programmer! :-)


  • The C Programming Language Kernighan & Ritchie


...but seriously, it is a good book, and a classic.  At the very least, check it out from your local library, and give it a look-through!

The Chapter structure is intended to ensure that exercises and documentation can be added in the future without disrupting the existing structure.  We will probably never actually provide material for all the topics listed there, even with the help of other collaborators that might join our project as time goes by.  But after redesigning the exercise structure and renaming exercise filenames several times (a tedious process), we realized that we need to have a general overview structure in place so that new material can be easily inserted.  This is the reason so many of the chapters are empty:  it is not that we are so ambitious, it is that we are tired of renaming files and directories over and over!
