What is a "function"?


For right now, think of function as a command you can type into your program that will do a specific job.


In this case, we are using the fwcDrawText() function, which draws some text on the graphical drawing area, which is the big box taking up approximately the top left third of the FeetWetCoding window at runtime (i.e. when the program you wrote in the Qt Creator text editor actually runs.)  Your graphical drawing area looks like this:



You can think of a function as a way of getting a bunch of work done in a single command.


Every function has a unique name.


The name of a function is case-sensitive.  SomeFunction() is a different function than somefunction().


When we type a function into our program, we say that we are "calling" that function.  At runtime, the computer goes off and does the work associated with that function, and then returns back to our code, and runs the next line that occurs after the function is finished doing it's work.


Most functions take parameters.


What is a "parameter"?


A parameter is a way of passing information to a function.  The information can be numbers, letters, variables and other things that we will get to in later exercises.  For now, just think of parameters as information you give to a function to help it get it's work done.


Parameters are located between the parentheses immediately after the function name, and are separated by commas.


Here is an example of a comment line, followed by the fwcDrawText() function with some parameters:


       // USAGE: fwcDrawText("Text.", x, y, color, fontsize);
       fwcDrawText("Change this text to...", 50, 150, BLUE, 15);


Remember the discussion of comments from the last exercise?  So the // characters at the beginning of the first line means that line is not code.  It is just a handy little human-readable note describing some information about the code.  This particular comment is usage information, describing how to use the fwcDrawText() function, and it explains that fwcDrawText() (which is called on the next line in the box above) takes five parameters:


● 1st Parameter:  a string of text, indicated by the " double-quotes around the text we want to draw on the screen.  In this case, the text we are drawing on the screen is "Change this text to..."

● 2nd Parameter: x, a number which tells fwcDrawText() where on the horizontal x-axis we want to locate the text on the screen.

● 3rd Parameter: y, a number which tells fwcDrawText() where on the vertical y-axis we want to locate the text on the screen.

● 4th Parameter: color, which tells fwcDrawText() what color we want the text to be.

● 5th Parameter: fontsize, which tells fwcDrawText() what size font we want the text to be drawn as.


These parameter names (x, y, color, fontsize) are named for our convenience.  We will usually use the same name in the USAGE comment that we actually named the parameter in the code.  We will cover this subject in later exercises.


Pay close attention every time you see a USAGE (or any other) comment in the code.  The USAGE comments are especially important, because they indicate what the parameters DO for the function they are describing!


So this is a pretty simple program:  really it is just one line, as the comment doesn't count as code!  Go ahead and run this exercise, and you will see that your exercise code on the left side of the runtime window is drawing the text Change this text to... , while you can see that on the solution (right) side of the runtime window, the text is  ...your name! :-)


The goal in these exercises is to make the left side of the window (where your code runs) match the right side of the window (where the solution code runs) however in this exercise, we want to change "Change this text to..." to whatever your name is, so you can see how to write text to the screen.


Alright!  Your first programming task!  Close the running FeetWetCoding program like you would close any program by hitting the X button on it's window.  Notice you do not want to close Qt Creator, where you edit your code in the FeetWetCoding project, only close the running FWC program.


Then just go and change the string (we call a series of characters like letters and numbers a "string") to your name, and hit the green arrow in Qt Creator to run the program again with the change you just made.  Keep the double-quotes, which tells C++ that this is a text string.  So for my name, I would change that line to:



       fwcDrawText("Robert", 50, 150, BLUE, 15);

You may notice that the name of the draw text function starts with "fwc" which stands for feet wet coding.  All of the names of the drawing tools you will use start with "fwc" because they were created specifically for this learning software.  We name them this way because we want to make it clear from the beginning that these tools are not part of standard C++.  C++ does not, by itself, have tools for drawing in a window.

You will recognize our simplified drawing tools because they start with "fwc" and you should understand that these "fwc" tools will not work except within the FeetWetCoding learning software.  They are not for "real world" programming, but are just here to help you learn.

Since C++ does not provide them, drawing tools are provided by graphical libraries, like Qt.  (Qt actually provides a lot more than just drawing tools, but for now, you can think of Qt as a graphical tools library.)  But the drawing tools that Qt (or any other graphical toolkit library) provides are not something that is easy for the beginning programmer to use.  The "right" way to do graphics is to directly use Qt or OpenGL or wxwidgets or DirectX or whatever graphical toolkit you like, but it will be awhile before you are able to do that, since you are just starting out.

This is a big reason why FeetWetCoding was created.  We have tried to improve the learning process (and make it a little more fun) by creating a small number of simplified drawing tools for you to use.  Graphics don't just make your programs more fun, they also make it visually obvious when your program is working correctly, and even better, graphics make it visually obvious when your program is not working correctly!  We believe that using graphics can help you learn.

Later, after you have more experience, you will be able to use the Qt drawing tools directly, but for now, the fwc tools should be plenty for you to learn the basics of programming in C++ ...and have some fun with graphics too!
