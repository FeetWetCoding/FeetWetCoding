Chapter 01, Exercise 02:  Text on the Screen


   //

   // FeetWetCoding Exercise 1:  How to display text in your program's window!

   //


   // In this exercise you will learn how to draw text in the drawing area.


   // We draw text on the screen with DrawText().  DrawText() is a "function".  When you see a word

   // with parentheses after it like that, it means that word is the name of a function.  We will talk

   // about functions more later.  For now, just think of a function as a tool that does a particular

   // job for you.


   // We have to tell DrawText() what text we want to draw, where to draw it, what font size to use,

   // and what color we want the text to be.  We put this information in between the parentheses

   // of DrawText() like this:


   // DrawText("Some text", x, y, fontsize, color);


   // ...we call the information between the () parentheses, "parameters".  Parameters are just

   // the information we give to the DrawText function in order to tell it what we want it to do.


   // The x and y parameters are used to determine where on the screen things are drawn.  x and y both

   // start at zero in the upper left corner of our window when the program runs.


   // X gets BIGGER towards the RIGHT edge of the window.

   // Y gets BIGGER towards the BOTTOM edge of the window.


   // We have to put the parameters in between the DrawText() parentheses in the same order

   // every time! It's always:


   // "text", x, y, fontsize, color


   // ...every time!  If you forget one, or type them in the wrong order, your program probably

   // won't work, and it may not even compile!  Be sure to read the HandsDirtyCPP/helpinfo

   // documentation for this exercise to learn what the compiler is and what to do if you get

   // compiler errors.


   DrawText("This is at location 5, 10 with font size 10 in black.",  5, 10, 10, BLACK);


   DrawText("This is at location 5, 30 with font size 13 in blue.", 5, 30, 13, BLUE);


   // Below, the "x coordinates", or the first column of numbers, are "stepping

   // on each other" so that each word is drawn over the next word.  Can you pick some

   // better numbers for the first column so that the sentence reads correctly and the words

   // are not on top of each other in a big mess?

   DrawText("Help!",        5,  100, 15, GREEN);

   DrawText("I'm",         50,  100, 15, GREEN);

   DrawText("all",         70,  100, 15, GREEN);

   DrawText("messed",      85, 100, 15, GREEN);

   DrawText("up!",         145, 100, 15, GREEN);


   // Can you fix this one too? :-)  (hint:  the numbers in the first column (the x column), starting

   // with 400 are in the wrong order.  DrawText for "This" should not have 300 there! :-)

   DrawText("This",        160, 200, 15, RED);

   DrawText("sentence",    210, 200, 15, RED);

   DrawText("is",          300, 200, 15, RED);

   DrawText("out",         5,  200, 15, RED);

   DrawText("of",          50, 200, 15, RED);

   DrawText("order!",      100, 200, 15, RED);


   // How about this one? :-)  (hint:  if you figured out the previous problems, this should be easy!)

   DrawText("Make",        5,  300, 8, PINK);

   DrawText("these",       40,  300, 10, PINK);

   DrawText("words",       80, 300, 12, PINK);

   DrawText("have",        130, 300, 14, PINK);

   DrawText("fontsize",    180, 300, 16, PINK);

   DrawText("10.",         260, 300, 18, PINK);


   // Here's some vertical text that is not lined up properly.  :-)

   // (hint:  you still need to change the x column values here, not the y column, which is set up

   // OK to have the words descend properly as the sentence is read downwards.)

   DrawText("Make",        70, 400,  15, GRAY);

   DrawText("these",       80, 425,  15, GRAY);

   DrawText("words",       60, 450,  15, GRAY);

   DrawText("line",        75, 475, 15, GRAY);

   DrawText("up",          65, 500, 15, GRAY);

   DrawText("vertically!", 60, 525, 15, GRAY);


   DrawReferenceBox(true, true);     // draw the graphical reference frame


   return 0;

}
