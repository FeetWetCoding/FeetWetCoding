Here are the goals of the FeetWetCoding learning framework:


  • The core design goal of FeetWetCoding is to get the person who is trying to learn C++ to start working with the compiler and actively changing source code from day one, rather than passively reading a textbook.  Textbooks on C++ are necessary to the learning process, and there are some great ones out there, but programming is not a passive activity.  Unlike many other disciplines, you can't really learn programming by reading alone:  you must get your hands dirty by modifying and running programs in order to learn how to program.
  • With FeetWetCoding, one of our primary goals is to ensure that once someone downloads our software package, they have all the necessary information they need to start coding within minutes.  We want to completely eliminate the confusing and overwhelming experience that too many beginners have to go through in googling forums and websites to just get their very first program to even compile and run!
  • FeetWetCoding tries to give the new programmer the ability to do some simple graphical programming early, which makes programming more fun and intuitive.  These days, in order to draw anything on the screen, you must have some idea of how to use classes and how event-driven programming works.  This is a real shame for the beginner, because in the early days of personal computing, the beginning programmer was able to have the experience of drawing on the screen in the first day or two of coding.  The graphics may have been crude, and the language (BASIC) may have been limited, but it sure was a lot more fun to learn that way than with all this "cin" and "cout" drudgery!  Learning to program by using cout is not nearly as fun as even the simplest graphical programs!
  • FeetWetCoding will always be an Open Source project, and will always be not only "free as in freedom", but "free as in beer" for individuals who want to learn C++.
  • FeetWetCoding intends to help beginning programmers learn C and C++ from within the QtCreator (pronounced "cute creator") SDK (Software Developer's Kit) using the Qt (pronounced "cute") programming library.  We chose Qt and QtCreator because they are free to individuals, and QtCreator SDK spares the beginning programmer the overwhelming task of mucking about trying to get WxWidgets or some other rendering library set up in the build toolchain, before they can even write "Hello World"!  Qt works well on Windows PCs, Macs and Linux boxes.  Qt and Qt Creator have a large user base, and the documentation for Qt is excellent.  Qt and QtCreator solve some annoying problems for us, and they do it elegantly and free of charge.
  • Here in North America, the pronunciation of Qt as "cute" does sound a little odd and, well, cutesy, to the ear of many engineers and programmers.  But the Qt people wrote it, and they get to decide what it is called and how to pronounce it, and that is how they pronounce it!  :-)  And anyways, for many people, pronouncing it as "que tee" means QT or Quick Time.  Don't confuse Qt with QT, they have nothing to do with each other.


Here is what FeetWetCoding is NOT:


  • FeetWetCoding is not a way to learn Python, Java, WxWidgets, OpenGL, or any other language or software library besides C++/Qt.  We (or someone else) may expand the FeetWetCoding concept in the future to introduce other languages and libraries, but that does not exist at the time this document was written.
  • FeetWetCoding does not work with Eclipse, NetBeans, VisualStudio, or any other development environment besides Qt and QtCreator.  Again, we may try to adapt FeetWetCoding to other build environments and tools at some point, but there is no plan to even start to think about that at the current time.
  • FeetWetCoding is NOT a complete, official, academic style textbook way to learn C++!  You will need to get a good C++ textbook, either one that we recommend, or another one.  If you intend to make programming your career, you will benefit greatly from proper academic instruction.  FeetWetCoding does not, and never will claim to be a complete, authoritative, formal way to learn C++.  We try to teach the right things the right way, and it is a core goal for us that we never teach bad habits.  If you are an experienced programmer and you see something in the exercises that you think is teaching bad programming habits, your feedback is welcome.  We may have reasons for doing things the way we are doing them, but we will always listen to feedback on the exercises.  We sincerely want to get those exercises as right as we can.
