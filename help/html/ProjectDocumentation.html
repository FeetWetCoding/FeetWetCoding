A Note About This Documentation


This documentation is currently generated with the helpndoc app.  We are considering putting it up on a wiki so that:


  • ...the latest documentation is easily available online.
  • ...people can contribute and collaborate with each other about the project.


However, the way we are doing the documentation now, by generating html with helpndoc and including that in the source tree under FeetWetCoding/help does allow us to include the complete documentation within the zip file that our users will download.  We feel that is important, but maybe that is only because we are older, and still make a distinction between "online" and "offline."


There is another issue, which is that I had a personal wiki up on another host, and did not take the time to learn how to properly manage it, and spammers came in and started using it for their SEO nonsense.  I don't want to have to deal with that on this project--there is already more than I can get done!  :-)


Finally, doing the documentation with helpndoc allows me to have complete control over it, whereas with a wiki, I might have to spend time every day (down the road) moderating contributions to the wiki.  If this project is going to grow, I have to learn to manage my time, and not waste time on overhead wherever possible.


So for now, this documentation is helpndoc-based.  But in the future we might move to some sort of online form of documentation, whether wiki, dokuwiki, tikiwiki or something else.  (tried Joomla... too many clicks to get anything done!  :-)


If we go to something like tikiwiki, we would still want to be able to generate the docs from the wiki, and include them in the .zip file that can be downloaded from github.  We welcome any feedback on this!
