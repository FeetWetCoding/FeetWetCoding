// Let's draw some things on the screen.  FeetWetCoding gives you some simple, easy to use

   // tools to draw on the screen.  These are called FUNCTIONS.  A function is like a tool

   // you can use to do work in your program.  There are predefined functions that you can

   // "call" by typing them into your program, and you can also define your own functions

   // to do particular tasks for you that you need to do again and again.  We will explore

   // functions and how they work, and how to design them in later exercises.


   // Here are some predefined graphical functions that HandsDirtyCPP gives you to work with:


   //  DrawLine( int xStart, int yStart, int xEnd, int yEnd, int thickness, Color color );

   //  DrawCircle( int x, int y, int r, int thickness, Color color, bool solid=false);

   //  DrawEllipse( int x, int y, int w, int h, int thickness, Color color, bool solid=false );

   //  DrawRectangle( int x, int y, int w, int h, int thickness, Color color, bool solid=false );

   //  DrawText( std::string text, int x, int y, int size, Color color );


   // We are going to draw some things on the screen so you can familiarize yourself with these

   // drawing functions and how they work.  The following code will draw a bunch of lines, circles,

   // ellipses, and rectangles of different sizes, colors and line widths on the left side of the

   // drawing area.  Your task is to change the code below to make graphics on the left side of

   // the screen look like the graphics on the right side.  The right side of the screen is not

   // drawn using the shape-drawing functions, but instead is a .bmp file containing a screenshot

   // of what the solution should look like.  This .bmp file is displayed in the drawing area

   // using the DrawImage function.  We will explain how to use DrawImage in a later exercise.


   // The first set of shapes, the drawn lines, may be the most difficult to get right.  If you get

   // stuck on the first section, try working with the other sections first:  the DrawCircle,

   // DrawEllipse and DrawRectangle sections.  This will help you familiarize yourself with the

   // x,y coordinates, as well as help you get used to how the different function parameters work.


   // Don't get discouraged!  Even experienced coders might not get every little detail exactly

   // right on their first try!  So it is perfectly normal for a beginning programmer to have to

   // make many tries at modifying this code in order to arrive at the solution.
