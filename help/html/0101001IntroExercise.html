Chapter 01, Exercise 01:  Intro Exercise




Welcome to your first C++ program!


NOTE: You NEED to read the help page How to Use FeetWetCoding to understand how to use the FeetWetCoding exercises!   And if you haven't already installed Qt Creator, you NEED to read the Install QtCreator help page and follow the instructions there or you will not be able to use FeetWetCoding at all.   If you have not read those pages and completed the instructions there,  please do so now.  Don't worry:  I'll wait here.  :-)




When you are done looking at the running program, don't forget to click the 'X' of the window of the running program to close it!  Once your program is running, if you make any changes to its code in Qt Creator,  the green "start" arrow button will not work again until you first close the program that is already running.  This will come up again:  I do it all the time!


So, to repeat, this process works:


  1. Edit your program.
  2. Run your program.
  3. Close your program window (NOT the Qt Creator window!)
  4. Go back to step 1.


...but this process does NOT work:


  1. Edit your program.
  2. Run your program.
  3. Try to go back to step 1 without first closing your running program.


When you forget to close the running program, and then you edit it and try to run it again, you will get this error in the Issues pane of the Qt Creator window:


:-1: error: cannot open output file debug\FeetWetCoding.exe: Permission denied


This is almost guaranteed to happen to you in the future, so if you changed something in your program and suddenly it won't run at all, this is the first thing you need to check:  did you forget to close the running program before making new changes?


For the rest of this exercise you will uncomment lines one at a time, and then compile and run the program so that you can clearly understand what each line does.  Remember to only uncomment lines which are not obviously English sentences!  If you uncomment English (or any other spoken language) sentences in a program, the program will not work.  The day may come when we can tell computers what to do by speaking English to them, but that day is not here yet.  :-)


Now, uncomment the first line of C++ code--NOT the first line in the file!  Uncomment the first DrawLine starting  with "//    DrawLine" by removing  the first two // characters, and compile and run the program to see what it does.  Change this:


//    DrawLine(50,0,50,100,3,BLUE);


...to this:


   DrawLine(50,0,50,100,3,BLUE);


...and then run the program by clicking the "Run" button (the green arrow.)  Did you have any idea what was going to be drawn on the screen?  It is OK if you didn't guess beforehand, but as you go, if you want to, you can try to see why the numbers between the parentheses make the shapes appear on the screen in the way they do.


Now uncomment the next "Draw..." line and run, and repeat for each subsequent line, one-at-a-time:


//    DrawCircle(300,100,100,3,RED, false);

//    DrawCircle(350,150,100,3,BLUE,true);


//    DrawCircle(550,150,100,3,BLUE,true);

//    DrawCircle(500,100,100,3,YELLOW,false);


//    DrawRectangle(50,250,700,300,3,BLACK,true);

//    DrawEllipse(400,400,30,70,3,BLUE);

//    DrawEllipse(350,400,30,70,3,YELLOW,false);

//    DrawEllipse(450,400,30,70,3,RED,true);

//    DrawRectangle(100,300,600,200,3,RED);


//    DrawLine(0,0,800,600,1,PINK);

//    DrawLine(0,600,800,0,1,GREEN);


All the circles and lines and so on are finally being drawn on the screen.  The last thing you will do in this exercise is:


   // Now that all the shapes are being drawn, comment out the following line of code so that you

   // can see exactly what DrawReferenceBox does by adding // at the beginning of the line:


   DrawReferenceBox(false, false);     // draw the graphical reference frame


   // Now you can check out DrawingAreaIntroSoln to make sure you got it right!


Remember:  you comment out a line of code by adding the // characters to the beginning of that line.


Congratulations!  You have completed your first FeetWetCoding exercise.  Change IntroExercise to IntroExerciseSoln in main.cpp, and check out the file a0005soln.cpp and you will be able to compare your work with the exercise solution!


Also, you are free to change the numbers in the () of each line, and re-run your program to see what change it might make in how the shapes get drawn on the screen.  If you make changes to the point where the program won't run anymore, you can restore a0005.cpp to it's original form by copying it from the exercises/orig directory.  exercises/orig contains all the exercise .cpp files in their original form, so that you can start over if you make changes to your program that you want to completely undo.


When you are ready, you can proceed to Ex 01-02: Text on the Screen.
