Release Checklist

Project Documentation ››

Print this out and check 'em off for every master release so we don't forget any steps:


√

Task


Create Version Release Issue in google code project.


In google code, pull up all "Fixed" issues and verify that they have been implemented correctly.


Edit helpndoc Version Information Page.


Change version number in both README.html files, the file LICENSE.txt, and in the helpndoc LICENSE page.


Generate documentation for distro.


Verify documentation coherence and copy the .hnd file to google docs.


Zip help/html dir, xfer up to feetwetcoding.com, unzip, and verify changes on pages.

(in chrome options > Under the Hood > Content Settings > All cookies and site data

then search on feetwet and blow out the entries for feetwetcoding.com)


Ensure github dev branch has all latest changes for dev from both our machines.


Pull down distro from dev branch ZIP button and build from scratch.


Verify functionality of changes, spot check core functionality.


Merge dev into master locally.


Push master up to github, and fetch/merge on both machines.


Pull distro down from master branch ZIP button and build from scratch on Windows, Mac & Linux.  On the Mac, reduce system screen resolution to 1024x768 to verify we are good at that rez.


Verify functionality of changes, spot check core functionality.


Switch back to dev branch on both machines.


Publish new release notice on feetwetcoding.com front page, google+ project stream, facebook stream, google groups.


Upload helpndoc file to google docs.


Close Version Release Issue in google code project.
