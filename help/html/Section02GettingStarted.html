Getting Started


Welcome to FeetWetCoding (fwc)!  This section, "Getting Started" introduces you to the basics of how you will get information from your program onto the screen.  Traditionally, if you started to learn C++, you would get your program's output by using something called "cout" which limits you to using text output, and does not allow you to do any graphics.  With fwc, you will be able to get output using something very similar that we have setup for you to use, called "seeout."  You will also be able to draw simple graphics on the screen, which you will use in later sections to see what your program is doing in a visual way.


Most C++ texts start off with about 50 pages of discussion of types, variables and other fundamental language concepts.  There's a good reason they do that, but with FeetWetCoding, we want you to jump right in and get your feet wet!  So we start off by showing you how to actually make your program do something:  draw stuff on the screen.


Now it turns out that without types, variables and all that other jazz, you can't really do much just by drawing things on the screen with the tools you are going to learn in this section by themselves.  But you will soon learn the C++ fundamentals, and then you will already know how to at least display information in graphical form that you will produce using these new tools and ideas.  Using graphics in this way will help give you a better visual sense of what your program is trying to do.


In this Section, you will learn:


● How to use the fwc drawing tool functions so that you can draw on the screen.  Later on if you forget how these drawing tools work, you can refer back to these exercises.

● How to use seeout so that your programs can easily display text.

● You will begin to develop some familiarity with how the x,y coordinate system is laid out.  You will do this by changing coordinates for where things are drawn in your exercise code so that your drawings match what the solution code produces.  This will start to make you more comfortable in understanding how to draw things on the screen in the places you want them, which is an important skill to have for all the remaining exercises you will be doing in FeetWetCoding.  By the end of this section, you should have an intuitive feeling for the difference between where 10,20 is on the screen and where 380,290 is, for example.
